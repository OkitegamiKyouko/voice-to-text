# Голос в текст

Небольшое браузерное приложение для расшифровки аудиозаписей с разделением реплик по голосам собеседников.

## Запуск

1. Откройте `index.html` двойным щелчком в современном браузере.
2. Выберите OpenAI или Google Gemini и вставьте соответствующий API-ключ. Ключ используется только в памяти вкладки и не сохраняется.
3. Загрузите ровно две аудиозаписи одного разговора.
4. Нажмите **Транскрибировать и объединить**.

## Загрузка в Google Drive

После обработки приложение может загрузить в выбранную папку Drive три файла: полную транскрипцию TXT, пересказ и выдержки TXT, а также JSON с говорящими и таймкодами. Загрузка использует Google Identity Services и узкий OAuth scope `drive.file`; токен живёт только в текущей вкладке.

1. В Google Cloud включите Google Drive API и создайте OAuth Client ID типа **Web application**.
2. Добавьте origin GitHub Pages `https://okitegamikyouko.github.io` в **Authorized JavaScript origins**. Путь `/voice-to-text/` в origin не добавляется.
3. Вставьте Client ID в блок Google Drive под готовой транскрипцией.
4. Оставьте заданный ID папки `Voice to Text` или укажите другую папку и нажмите **Подключить Drive и загрузить**.

Client ID не является секретом и сохраняется только в `localStorage` браузера. Приложение не хранит access token после закрытия или обновления вкладки.

В режиме OpenAI приложение использует `gpt-4o-transcribe-diarize` для распознавания и `gpt-5.4-mini` для объединения версий. В режиме Google все этапы выполняет `gemini-3.5-flash`. При временной перегрузке Gemini приложение повторяет запросы с увеличивающейся задержкой, а затем автоматически переключается на `gemini-3.1-flash-lite`. Для работы нужны интернет и активный ключ выбранного API. Использование API оплачивается отдельно от пользовательских подписок ChatGPT или Gemini.

Приложение сначала транскрибирует обе записи отдельно, а затем выбранная нейросеть сопоставляет версии, убирает дубли и артефакты распознавания и формирует единый чистовой текст. После этого она готовит краткий пересказ и выбирает несколько дословных ключевых выдержек с привязкой к записи и времени. Уникальные реплики сохраняются. Результат можно скачать одним TXT-файлом.

Из-за лимита inline-запроса браузерный режим Gemini принимает каждый аудиофайл до 14 МБ. Запас нужен для Base64-кодирования и промпта внутри общего лимита запроса. OpenAI принимает каждый файл до 25 МБ. Для более крупных файлов понадобится небольшой серверный компонент с Gemini Files API.

Если браузер блокирует запрос из локального файла, запустите любой статический HTTP-сервер в этой папке, например через расширение Live Server в VS Code.

## Возможности

- загрузка двух файлов перетаскиванием;
- встроенное прослушивание обеих записей;
- автоматическое определение языка или ручной выбор;
- отдельная транскрибация и автоматическая дистилляция двух версий;
- приблизительная оценка оставшегося времени по длительности аудио и текущему этапу;
- краткий пересказ и ключевые дословные выдержки после транскрипции;
- временные метки, источник записи и разделение по голосам;
- переход к реплике по клику;
- переименование собеседников;
- копирование и экспорт в TXT/JSON.

## Конфиденциальность

В проекте нет собственного сервера и базы данных. Файлы передаются из браузера напрямую в API выбранного провайдера после явного нажатия кнопки. Перед использованием с конфиденциальными записями проверьте применимые требования и политику обработки данных вашей организации.
